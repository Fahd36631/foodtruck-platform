# 🚀 Stage 4 — MVP Development and Execution

## 📋 Project Overview

This stage focused on implementing and executing the **MVP (Minimum Viable Product)** for our food truck platform project.

The team followed **Agile-style collaboration** during development by:
- Dividing responsibilities across team members
- Reviewing progress continuously
- Fixing issues iteratively
- Improving the MVP based on testing and mentor feedback

> ⚠️ The frontend environment was developed locally using **Expo Go**, as discussed and approved with the mentor during the MVP phase.

## 🔄 Development Workflow

The team followed an **iterative MVP development approach**:

```
1. Plan feature implementation
2. Develop and integrate features
3. Review Pull Requests
4. Perform QA testing
5. Fix issues and improve functionality
6. Deploy backend updates to production
```
